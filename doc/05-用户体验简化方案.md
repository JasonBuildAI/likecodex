## 一、当前用户体验问题

### 1.1 从零到可用的完整路径

```
Step 1:  意识到需要安装 Rust 工具链                     ❓ 为什么 coding agent 需要 Rust？
Step 2:  访问 rustup.rs 安装 Rust                        ⏱ 3-5 分钟
Step 3:  安装 Python 3.11+                               ⏱ 2-3 分钟
Step 4:  安装 uv (pip install uv 或官网脚本)              ⏱ 1 分钟
Step 5:  安装 Node.js 20+                                ⏱ 2-3 分钟
Step 6:  git clone 项目                                  ⏱ 1 分钟
Step 7:  uv sync --all-packages                          ⏱ 2-5 分钟
Step 8:  cargo install --path crates/likecodex-cli        ⏱ 3-10 分钟
Step 9:  cd web && npm install --legacy-peer-deps         ⏱ 1-3 分钟
Step 10: 配置 .env 文件                                   ⏱ 1 分钟
Step 11: 配置 ~/.likecodex/config.toml                    ⏱ 1 分钟
Step 12: 启动 likecodex start --web                       ⏱ 3-5 秒
────────────────────────────────────────────────────
总计: ≈25 分钟，12 步，4 种工具链
```

### 1.2 对比目标

| 指标 | 当前 | 目标 |
|------|------|------|
| 安装步骤 | 12 步 | **1 步** |
| 安装耗时 | 15-25 分钟 | **30 秒** |
| 依赖工具链 | Rust + Python + Node.js + uv | **Python + pip** |
| 配置项 | 10+ 环境变量 + TOML | **交互式问答 3 项** |
| 新手引导 | 无 | **内置引导** |

### 2.2 交互式配置向导（P0）

**新建文件**: `likecodex_engine/setup.py`

```python
"""交互式配置向导"""
import os
import shutil
from pathlib import Path
from rich.console import Console
from rich.prompt import Prompt, Confirm, Select
from rich.panel import Panel
from rich.markdown import Markdown

console = Console()

CONFIG_TEMPLATE = """# LikeCodex 配置文件
# 自动生成于 {date}

[llm]
provider = "deepseek"
model = "{model}"
api_key = "{api_key}"

[approval]
mode = "{approval_mode}"

[agent]
enable_planner = {enable_planner}
"""

WELCOME = """# 🎉 欢迎使用 LikeCodex！

LikeCodex 是专为 DeepSeek V4 打造的 AI 编程助手。
让我帮你完成初始配置（约 2 分钟）。
"""

COMPLETION = """# ✅ 配置完成！

你可以这样开始使用：

  1. 交互模式:  likecodex --chat
  2. 单次任务:  likecodex "帮我检查这个项目的代码质量"
  3. Web 界面:  likecodex --web（自动打开浏览器）

需要帮助? 运行 likecodex --help
"""


async def interactive_setup():
    """完整的交互式配置流程"""
    
    console.print(Markdown(WELCOME))
    
    # Step 1: API Key
    console.print("\n[bold]Step 1/3: 配置 DeepSeek API Key[/bold]")
    console.print("  你可以在 https://platform.deepseek.com 获取 API Key")
    
    api_key = Prompt.ask(
        "请输入你的 DeepSeek API Key",
        password=True,
    )
    
    if not api_key.startswith("sk-"):
        console.print("[yellow]⚠ API Key 通常以 'sk-' 开头，请确认是否正确[/yellow]")
        if not Confirm.ask("继续?"):
            api_key = Prompt.ask("请重新输入 API Key", password=True)
    
    # 测试连接
    console.print("\n[bold]测试 API 连接...[/bold]")
    if await test_deepseek_connection(api_key):
        console.print("[green]✅ API 连接成功！[/green]")
    else:
        console.print("[red]❌ 连接失败，请检查 API Key 是否正确[/red]")
        if not Confirm.ask("忽略并继续?"):
            return
    
    # Step 2: 选择模型
    console.print("\n[bold]Step 2/3: 选择模型[/bold]")
    model = Select.ask(
        "选择默认模型",
        choices=[
            ("deepseek-v4-flash", "快速、经济（推荐日常使用）"),
            ("deepseek-v4-pro", "更强、更贵（适合复杂任务）"),
        ],
    )
    
    # Step 3: 审批模式
    console.print("\n[bold]Step 3/3: 选择审批模式[/bold]")
    approval_mode = Select.ask(
        "选择工具审批模式",
        choices=[
            ("auto", "自动审批低风险操作（推荐）"),
            ("ask", "每次操作都询问"),
            ("full-access", "完全信任，不审批"),
            ("read-only", "仅读取，不执行任何修改"),
        ],
    )
    
    # 生成配置
    config_dir = Path.home() / ".likecodex"
    config_dir.mkdir(parents=True, exist_ok=True)
    config_path = config_dir / "config.toml"
    
    config_content = CONFIG_TEMPLATE.format(
        date=__import__("datetime").datetime.now().strftime("%Y-%m-%d %H:%M"),
        model=model,
        api_key=api_key,
        approval_mode=approval_mode,
        enable_planner="true" if model == "deepseek-v4-flash" else "false",
    )
    
    config_path.write_text(config_content, encoding="utf-8")
    console.print(f"\n[green]✅ 配置已保存到 {config_path}[/green]")
    
    # 创建项目记忆文件
    if Confirm.ask("\n是否为当前项目创建记忆文件 (.likecodex/memory.md)？"):
        await create_project_memory()
    
    console.print(Markdown(COMPLETION))
```

### 2.3 一键安装脚本更新（P0）

根据方向一的架构决策，更新安装脚本。

**修改文件**: `scripts/install.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

# LikeCodex 安装脚本
# 用法: curl -fsSL https://get.likecodex.dev | bash

echo "🚀 正在安装 LikeCodex..."

# 检查 Python
if ! command -v python3 &> /dev/null; then
    echo "❌ 需要 Python >= 3.11"
    echo "   请先安装: https://www.python.org/downloads/"
    exit 1
fi

PYTHON_VERSION=$(python3 -c "import sys; print(f'{sys.version_info.major}.{sys.version_info.minor}')")
if [[ "$(echo -e "3.11\n$PYTHON_VERSION" | sort -V | head -n1)" != "3.11" ]]; then
    echo "❌ 需要 Python >= 3.11，当前版本: $PYTHON_VERSION"
    exit 1
fi

echo "📦 正在从 PyPI 安装 LikeCodex..."
pip install likecodex

# 运行配置向导
echo "⚙️ 运行初始配置..."
likecodex setup

echo ""
echo "✅ 安装完成！运行 likecodex --chat 开始使用。"
