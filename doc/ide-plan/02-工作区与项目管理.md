## 一、目标

构建 LikeCodex IDE 的工作区管理系统，实现对项目文件的浏览、监听、搜索和配置管理，为后续所有 IDE 功能提供基础。

### 1.1 核心能力

| 能力 | 优先级 | 说明 |
|------|--------|------|
| **文件树浏览** | P0 | 树形文件列表，支持展开/折叠/上下文菜单 |
| **文件 CRUD** | P0 | 创建/读取/更新/删除文件和目录 |
| **文件监听** | P0 | 实时检测文件变更（外部编辑器修改、git pull 等） |
| **多根工作区** | P1 | 同时打开多个目录 |
| **全局搜索** | P1 | 跨文件全文搜索（ripgrep 后端） |
| **项目配置** | P0 | `.likecodex/` 配置目录，支持规则、设置 |
| **最近项目** | P1 | 历史项目列表，快速打开 |
| **大纲视图** | P2 | 基于 tree-sitter 的符号大纲 |

### 1.2 对标 Cursor / VS Code

| 功能 | VS Code | Cursor | LikeCodex 目标 |
|------|---------|--------|---------------|
| 文件树 | Explorer 视图 | 同 VS Code | ✅ 自建文件树组件 |
| 文件监听 | FileSystemWatcher | 同 VS Code | ✅ 基于 Rust notify crate |
| 多根工作区 | 工作区文件 `.code-workspace` | 同 VS Code | ✅ `.likecodex-workspace` |
| 全局搜索 | Ctrl+Shift+F | 同 VS Code | ✅ 基于 ripgrep |

| 大纲 | Outline 视图 | 同 VS Code | ⏳ Phase 2 |

### 2.3 文件系统 API

**Rust 端** (`crates/likecodex-ide-fs/src/lib.rs`):

```rust
/// 高性能文件系统操作
pub struct FileSystemService {
    watcher: FileWatcher,
    ignore_rules: IgnoreRules,  // gitignore-aware
}

impl FileSystemService {
    /// 读取文件内容（UTF-8 或 Binary 检测）
    pub async fn read_file(&self, path: &Path) -> Result<FileContent>;
    
    /// 写入文件（自动创建父目录）
    pub async fn write_file(&self, path: &Path, content: &[u8]) -> Result<()>;
    
    /// 列出目录内容（gitignore 过滤）
    pub async fn list_dir(&self, path: &Path) -> Result<Vec<FileEntry>>;
    
    /// 创建目录（类似 mkdir -p）
    pub async fn create_dir(&self, path: &Path) -> Result<()>;
    
    /// 删除文件或目录（递归）
    pub async fn delete(&self, path: &Path) -> Result<()>;
    
    /// 移动/重命名
    pub async fn move_file(&self, from: &Path, to: &Path) -> Result<()>;
    
    /// Glob 搜索
    pub async fn glob(&self, pattern: &str) -> Result<Vec<PathBuf>>;
    
    /// 全文搜索（基于 ripgrep）
    pub async fn search_text(&self, query: &str, root: &Path) -> Result<Vec<SearchMatch>>;
}
```

**文件监听** (`crates/likecodex-ide-fs/src/watcher.rs`):

```rust
use notify::{Config, Event, EventKind, RecommendedWatcher, Watcher};
use tokio::sync::broadcast;

pub struct FileWatcher {
    tx: broadcast::Sender<FileEvent>,
    watcher: RecommendedWatcher,
}

#[derive(Clone, Debug)]
pub enum FileEvent {
    Created(String),
    Modified(String),
    Deleted(String),
    Renamed(String, String),  // from, to
}

impl FileWatcher {
    pub fn new(root: &Path) -> Result<(Self, broadcast::Receiver<FileEvent>)> {
        let (tx, rx) = broadcast::channel(256);
        let tx_clone = tx.clone();
        
        let mut watcher = RecommendedWatcher::new(
            move |res: Result<Event, notify::Error>| {
                if let Ok(event) = res {
                    // 映射 notify::Event 到 FileEvent 并发送
                    for path in &event.paths {
                        let _ = tx_clone.send(/* ... */);
                    }
                }
            },
            Config::default(),  // 默认递归监控
        )?;
        
        watcher.watch(root, RecursiveMode::Recursive)?;
        
        Ok((Self { tx, watcher }, rx))
    }
}
```

### 2.5 Zustand Store

```typescript
// web/src/stores/workspaceStore.ts
import { create } from 'zustand';

interface FileNode {
  path: string;
  name: string;
  extension: string;
  isDirectory: boolean;
  children?: FileNode[];
  expanded: boolean;
}

interface WorkspaceState {
  // 状态
  rootUri: string | null;
  files: FileNode[];
  recentProjects: ProjectInfo[];
  isOpen: boolean;
  fileEvents: FileEvent[];
  
  // 操作
  openWorkspace: (uri: string) => Promise<void>;
  closeWorkspace: () => void;
  refreshFileTree: () => Promise<void>;
  expandDirectory: (path: string) => Promise<void>;
  collapseDirectory: (path: string) => void;
  createFile: (path: string) => Promise<void>;
  createDirectory: (path: string) => Promise<void>;
  deleteFile: (path: string) => Promise<void>;
  renameFile: (oldPath: string, newPath: string) => Promise<void>;
  searchFiles: (query: string) => Promise<SearchResult[]>;
}

export const useWorkspaceStore = create<WorkspaceState>((set, get) => ({
  rootUri: null,
  files: [],
  recentProjects: [],
  isOpen: false,
  fileEvents: [],
  
  openWorkspace: async (uri: string) => {
    const res = await fetch(`/api/ide/workspace/open`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ rootUri: uri }),
    });
    const info = await res.json();
    set({ rootUri: uri, isOpen: true });
    // 连接文件事件 SSE
    connectFileEvents(uri);
    // 加载文件树
    get().refreshFileTree();
    // 保存到最近项目
    saveRecentProject(info);
  },
  
  refreshFileTree: async () => {
    const res = await fetch(`/api/ide/fs/list?path=`);
    const files = await res.json();
    set({ files });
  },
  
  expandDirectory: async (path: string) => {
    const res = await fetch(`/api/ide/fs/list?path=${encodeURIComponent(path)}`);
    const children = await res.json();
    set(state => ({
      files: updateNodeInTree(state.files, path, { expanded: true, children })
    }));
  },
  // ... 更多操作
}));
```

## 三、实现步骤

### 3.1 Phase 1：基础文件树（第 1 周）

**Rust 端**:
1. 创建 `crates/likecodex-ide-fs` crate，实现 `FileSystemService`
2. 在 `likecodex-server` 中注册 IDE 文件系统路由
3. 实现文件读取、写入、目录列表
4. 实现文件监听（notify crate）+ SSE 推送

**前端**:
1. 创建 `workspaceStore`（Zustand）
2. 创建 `FileTree` 组件（递归渲染、懒加载子目录）
3. 创建文件树上下文菜单（新建文件/目录、重命名、删除）
4. 实现文件双击在编辑器中打开（与文档 03 接口对接）

### 3.2 Phase 2：全局搜索 + Git 状态（第 2 周）

**Rust 端**:
1. 实现全文搜索（通过 `grep` crate 或调用 ripgrep）
2. 实现 Git 状态集成（对每个文件检测 Git 状态）
3. 实现 Glob 搜索 API

**前端**:
1. 创建搜索面板组件（搜索输入 + 结果列表）
2. 文件树节点显示 Git 状态徽标
3. 忽略文件模式配置（继承 `.gitignore`）

### 3.3 Phase 3：多根工作区 + 项目配置（第 3 周）

**Rust 端**:
1. 支持多根工作区配置
2. 实现 `.likecodex/config.toml` 读写 API
3. 实现 `.likecodex/rules/` 规则文件管理

**前端**:
1. 工作区选择器 UI（最近项目列表 + 浏览按钮）
2. 项目设置面板（UI 编辑 `.likecodex/config.toml`）
3. 大纲视图（基于 tree-sitter 的符号提取）
