## 一、愿景

将 LikeCodex 从 **AI Chatbox + 工具集** 改造为 **AI-Native IDE**，全面对标 Cursor 的核心能力，同时保留和强化 LikeCodex 的独特优势（DeepSeek 深度优化、Python-first 架构、开源可定制）。

### 1.1 目标状态

```
当前:  AI Chatbox + 30+ Agent Tools → 用户发送文本 → AI 返回代码 → 用户手动应用
目标:  AI-Native IDE                  → 用户可视化编辑 + AI 实时协作 → 自动应用修改
```

### 1.2 对标 Cursor 的核心能力矩阵

| 能力 | Cursor | LikeCodex 当前 | 改造目标 |
|------|--------|---------------|----------|
| **AI Chat (Cmd+L)** | ✅ | ✅ 已有 | 增强上下文引用 |
| **内联编辑 (Ctrl+K)** | ✅ | ❌ 缺失 | **新增** |
| **Tab 补全** | ✅ | ❌ 缺失 | **新增** |
| **Composer 多文件编辑** | ✅ | ⚠️ Agent 有但无 UI | **增强** |
| **代码库索引** | ✅ | ✅ 已有 CodeGraph | 增强为实时索引 |
| **Terminal AI** | ✅ | ❌ 缺失 | **新增** |
| **Debug 集成** | ✅ | ❌ 缺失 | **新增** |
| **Git 可视化** | ✅ | ❌ 缺失 | **新增** |
| **Diff 视图** | ✅ | ⚠️ 基础 Diff 有 | **增强** |
| **@ 上下文引用** | ✅ | ❌ 缺失 | **新增** |

### 2.2 关键架构决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| **桌面壳** | Tauri v2（保留现有） | 比 Electron 轻量 10x，Rust 生态一致 |
| **编辑器** | Monaco Editor（保留现有） | 业界标准，VS Code 同款，生态丰富 |
| **终端** | xterm.js + node-pty | 标准终端模拟方案，与 Monaco 集成良好 |
| **IDE 前端框架** | React + Zustand（保留现有） | 已在用，不需要重写 |
| **通信协议** | HTTP REST + SSE（保留现有） | 已在用，支持流式 |
| **进程模型** | 单 Tauri 进程 + Rust Server + Python Engine | 保留三层，增加进程健康管理 |
| **文件系统** | Rust 层提供高性能 FS 操作 | 利用 Rust 的跨平台 FS 能力 |
| **包管理** | `pip install likecodex`（保留） | 方向一的决策 |

## 五、技术风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| Monaco Editor 性能瓶颈 | 大文件卡顿 | 中 | 虚拟滚动 + 延迟加载 |
| Tauri 与 Monaco 集成问题 | 开发阻塞 | 低 | 先用 Web 模式开发 |
| Python AI 引擎延迟 | IDE 响应慢 | 中 | 预加载 + 缓存 + SSE 流式 |
| 多进程管理复杂 | 可靠性下降 | 中 | Tauri 侧进程管理器 + 健康检查 |
| 与现有 Agent 架构冲突 | 集成困难 | 低 | 分层设计，IDE 层调用 Agent API |
